ETL y Analisis de Datos (SQL+Power BI)

Enero de 2024

Resumen:

Este proyecto tuvo como objetivo principal, entregar información para el control de las asistencias de los trabajadores, la empresa para la que se desarrollo el proyecto es una de producción y exportación de Berries ubicada en el cantón Zapotillo de la provincia de Loja con alrededor de 400 trabajadores, los softwares que se utilizaron fueron Excel, Python, SQL Server y Power BI, el flujo de trabajo va desde la extracción de la información, limpieza de los datos y transformación, creación de una base de datos, carga en Power BI y elaboración del dashboard.


Fuente de Datos

La información se la obtiene principalmente del ERP AGRITRACER, software destinado a proporcionar información relevante sobre el registro de las horas laboradas por el trabajador en el sector agrícola esta información se descarga diariamente en formato xlsx y csv, también se utilizó información registrada en tablas elaboradas por el analista en función de los acontecimientos diarios.

Limpieza de Datos

La limpieza de los datos se realizó en Power Query de Microsoft Excel, para poder automatizar el proceso y por la facilidad y compatibilidad en lo formatos de las fuentes de datos, se utilizó instrucciones desde la interfaz y algunos códigos de lenguaje m propios, luego se procedió a cargas las tablas en hojas independientes que se transformaron en formato csv, de esta manera se pudieron cargar a SQL Server por medio de código, para completar la automatización del proceso.

Data Werehouse

El modelado de la base de datos se realizó en SQL Server. Se relacionó la tabla extraída del ERP con los detalles del trabajador, periodos en los que ha trabajado en la empresa (los trabajadores entran y salen recurrentemente debido al giro del negocio), ingresos diarios en el transporte (bus proporcionado por la empresa o transporte propio) para identificar asistencias diarias, actividades, con sus centros de costo y las áreas a las que pertenecen cada una, entre otros atributos.
